# Article Outline Patterns (decision-first + AEO)

Maps to the "Article Outline" stage in the flow diagram. Patterns and rules for designing an outline. Voice follows the Zynkr Brand Guide (decision-first).

## Rule 1: Answer up front (answer-first, for AEO)

The first paragraph (within 80–120 characters) directly answers the question in the title, so AI engines can quote the whole passage.

## Rule 2: Make the Method visible (for the brand)

The outline should make at least one move of the Zynkr Method visible: Frame (frame the real decision first) / Clarify / Constrain / Compare (call out the trade-off) / Commit (close on a clear direction). For decision-type articles, follow the "frame the decision → real trade-offs → give a direction" skeleton.

## Common outline patterns (pick by intent)

| Intent | Recommended pattern | Skeleton |
|---|---|---|
| Informational | concept → classification → application | Definition → why it matters → how to do it → examples |
| Commercial investigation | comparison / trade-off | Overview of options → real trade-off comparison table → breakdown of each option → how to choose |
| Transactional decision | decision framework | The decision you're making → the trade-off behind it → first-hand experience as evidence → clear direction (CTA) |

## What every H2 should specify

- A single key-point direction (not just a heading)
- Where the target / supporting keyword lands
- 3 writable key points (for content-draft)

## AEO structural components (include as needed)

- FAQ block (3–5 questions, for schema)
- Comparison table (mandatory for commercial-investigation intent; used to present the trade-off)
- Steps / checklist (can be extracted as a list)
- One "named framework" — ideally naming a decision framework / trade-off
