# Subtitle Rewrite Rules

以下規則用於把逐字轉錄字幕整理成可直接上影片的繁體中文字幕。

直接輸出成果，不要加入招呼或說明。

## 最重要原則

- 高度貼近原始 `.srt` 內容。
- 預設只能做最小修改，不可自由改寫。
- 可以動的只有三類：
- 刪除冗餘語氣詞。
- 修正常見或明顯的 STT 錯字。
- 依字幕閱讀需求調整換行、空格與少量字面整理。

以下行為預設禁止：
- 壓縮資訊。
- 改寫成更精煉的說法。
- 把兩句意思合併成一句。
- 把具體說法改成摘要式說法。
- 因為覺得更順就自行換詞。

## 主要原則

1. 為了影片閱讀節奏，每一行不超過 18 個中文字。
2. 換行時每一行要盡量完整句，自然不突兀。
3. 去除所有標點符號。
4. 當一行超過 10 個字但還不需要換行時，可以在語氣合理處加入半形空格。
5. 中文遇到英文或數字時，前後加半形空格，例如：`我有 3 台 iPhone 手機`
6. 保留專業術語英文正確寫法與縮寫，例如：`FLAC`、`Notion`、`n8n`、`Claude Code`
7. 可以刪除冗餘無意義語氣詞，但保留講者情緒與原本資訊密度。
8. 如果講的是 AI 或 LLM，不要用「他」或「她」，改用「它」。
9. 如果口誤或重複講詞，字幕要校正成精準易懂版本，但不能改變原意，也不能順手摘要。

## 常見可刪語氣詞

- 那
- 欸
- 哈哈哈
- 對吧
- 嘛
- 然後

## 常見轉錄錯誤字辭典

- `Cloud 公司` -> `Claude 公司`
- `no 選` -> `Notion`
- `cloud upper` -> `Claude Opus`
- `克制` -> `客製`
- `AI 寫作客` -> `AI 寫作課`
- `見徑式` -> `漸進式`
- `antropic` -> `Anthropic`
- `jammernike` -> `Gemini`
- `大於模型` -> `大語模型`
- `prump` -> `prompt`
- `gbt` -> `GPT`
- `中端` -> `終端`

## 額外約束

- 保持字幕可對回原影片，不要改 cue 順序或時間碼。
- 優先逐字貼近，其次才是可讀性優化。
- 若有不確定內容，保守修正，不要補出原音沒有的資訊。
- 若某一段不是明顯錯字或語氣詞，就保留原句。
